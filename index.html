Problem Statement

Given Spotify's audio feature data — danceability, energy, loudness, tempo, speechiness, and more — can we reliably predict whether a track will be popular? This project converts the continuous track_popularity score into a binary classification target using the dataset median as a threshold, then applies logistic regression to learn the relationship between audio features and popularity.

Songs scoring at or above the median are labeled Popular (1); those below are Not Popular (0). This framing makes the problem tractable with logistic regression, which outputs a probability between 0 and 1 via the sigmoid function.

02 — Methodology

Analysis Pipeline

01

Exploratory Data Analysis

Visualized distributions and correlations across all continuous and categorical features using bar charts, heatmaps, box plots, and scatter plots. Identified strong positive correlation between energy and loudness, and a negative trend between both energy/duration and track popularity.

02

Unsupervised Clustering

Applied both KMeans and Hierarchical Agglomerative Clustering on standardized audio features. Used elbow method and dendrogram analysis to determine the optimal cluster count. Both methods converged on 6 clusters — matching the 6 distinct playlist genres in the dataset, suggesting genre is a natural grouping dimension.

03

Model Specification & Fitting

Built three logistic regression formulas with increasing complexity: an additive model (28 coefficients), a model with select interactions (56 coefficients), and a higher-order interaction model (198 coefficients). Fit using statsmodels for statistical interpretability (p-values, confidence intervals) and scikit-learn for pipeline integration.

04

Prediction & Evaluation

Generated predictions on the training set and evaluated using confusion matrices, ROC curves, and AUC scores. Examined coefficient magnitudes and statistical significance to identify the most influential features. Probability threshold set at 0.50 for class assignment.

05

Stratified Cross-Validation

Used StratifiedKFold with a full scikit-learn Pipeline (StandardScaler + OneHotEncoder via ColumnTransformer + LogisticRegression) to evaluate generalization performance. Cross-validation revealed that the simplest model generalized best, while the two complex models exhibited overfitting.

03 — Key Findings

What the Data Revealed

🎸

Genre is the Strongest Signal

All six playlist_genre categories were statistically significant across models. Rock and Pop exerted the largest coefficients, aligning with the clustering result of 6 natural song groupings that mapped directly onto genre.

⚡

Energy Hurts Popularity

Despite energy and loudness being highly positively correlated with each other, they had opposite signs in the model: energy's coefficient was negative, while loudness was positive. Higher-energy tracks tended to be less popular in this dataset.

⏱️

Longer Songs = Less Popular

duration_ms showed a consistent negative relationship with popularity. As track length increased, the probability of being classified as popular decreased across all fitted models.

🗂️

Clustering Validates Modeling

KMeans and Hierarchical Clustering independently identified 6 optimal clusters from continuous audio features alone — the same number as the genre categories. This unsupervised result reinforces that genre is a meaningful, latent structure in the data.

🎤

Cluster Specialization

Cluster 0 dominated on speechiness (high spoken-word content), while Cluster 1 dominated on liveness (live performance characteristics). These aligned intuitively with genres like R&B/rap and latin, respectively.

📉

Simplicity Wins

The simplest model (28 coefficients) achieved the best cross-validated test accuracy. The gap between training and test performance grew as model complexity increased, a textbook demonstration of the bias-variance tradeoff.

Supplemental Analysis

Cross-Validation Deep Dive Notebook

A supplemental notebook (Part_F_Cross_Validation.ipynb) was built to explore and compare three distinct approaches to stratified CV, isolating the effect of framework and bookkeeping choices on results.

🧮

Method 1 — statsmodels + Manual Preprocessing

StandardScaler and OneHotEncoder applied manually inside each fold. Formula strings updated to match encoded column names. Used in the final notebook.

🔧

Method 2 — sklearn Pipeline (Wide Output)

Patsy design matrices fed into a ColumnTransformer pipeline — numerics scaled, dummies passed through. Train and test scores stored in separate columns.

📐

Method 3 — sklearn Pipeline (Long Output)

Same pipeline as Method 2 with results stored in long format using a data_split column — directly compatible with seaborn catplot.

All three methods confirmed the same conclusion: Model 0 (28 coefficients) generalizes best. The work also surfaced a discrepancy in Model 2's train/test behavior across methods — a meaningful observation about how patsy interaction terms interact with pipeline preprocessing.
